v1.1.1 - Release Stable GA Promotion and Version Consistency

## Summary

Release v1.1.1 promotes all 21 GitHub releases (v0.1.0 through v1.1.0) to stable/GA status by unchecking the "This is a pre-release" designation. This release also resolves version-string inconsistencies across the repository, ensuring that `pyproject.toml`, `CITATION.cff`, `README.md` badge, `README.md` bibtex citation, `federated/__init__.py`, and `docs/repository_summaries.md` all report the same version. No functional code changes are included — this is a release metadata and documentation consistency update.

## Features

- **GitHub Release Promotion** — All 20 pre-release versions promoted to stable by unchecking "This is a pre-release" on each GitHub Release page. v1.1.0 was already stable. The repository now presents a consistent stable release history.

- **Strict Version-String Consistency** — Every file that declares a version now reports `1.1.1`. Previously, `federated/__init__.py` (0.2.0), `README.md` bibtex (1.0.1), and `docs/repository_summaries.md` (1.0.1) lagged behind the canonical version.

- **Repository Maturity Metadata** — `pyproject.toml` classifier `Development Status :: 5 - Production/Stable` confirmed. No pre-release or beta language remains anywhere in the repository's release artifacts.

- **Prompt 19** — Full v1.1.1 stable GA promotion prompt documented in `prompts.md`.
