# v0.9.5 - Peer Review Fixes: CI Determinism and Process Hardening

## Summary

This release implements all 12 code-fix recommendations from the v0.9.4 senior engineering peer review. The primary focus is eliminating CI lint flake risk by pinning tool versions, adding local reproducibility tooling, aligning version metadata across the project, and hardening the contributor workflow with pre-commit hooks and a lint script that mirrors CI exactly.

## Features

- **CI Determinism (P0):** Lint tools now installed via `constraints/lint.txt` with pinned versions (`ruff==0.15.1`, `yamllint==1.38.0`), eliminating non-deterministic behavior from floating PyPI versions. A new "Show tool versions" diagnostic step in CI prints Python, ruff, and yamllint versions plus `pip freeze` output for auditability.
- **Constraints File (P1):** `constraints/lint.txt` serves as the single source of truth for lint tool versions, referenced by CI (`.github/workflows/ci.yml`), local lint script (`scripts/lint.sh`), and pre-commit (`.pre-commit-config.yaml`).
- **Local Lint Reproducibility (P1):** `scripts/lint.sh` mirrors CI lint commands exactly — developers can run one command to validate locally before pushing. Supports `--fix` flag for auto-correction.
- **Pre-commit Hooks (P2):** `.pre-commit-config.yaml` provides ruff check, ruff format, and yamllint hooks pinned to the same versions as CI.
- **Version Alignment (P2):** Synchronized version `0.9.5` across `pyproject.toml`, `CITATION.cff`, `README.md` badge, `CHANGELOG.md`, and `releases.md`.
- **Peer Review Traceability:** `peer-review/v0.9.5-peer-review-fixes.md` documents each of the 12 fixes with file-level traceability, correction process metrics, and final validation results.
