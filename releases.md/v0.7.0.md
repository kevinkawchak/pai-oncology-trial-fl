# v0.7.0 — Security, Logic, and Compliance Audit

## Summary

Comprehensive security, logic, and compliance audit of all 82 Python files across the entire repository. Identified and remediated 61 findings: 12 security vulnerabilities (including 5 arbitrary code execution via `torch.load`), 14 logic bugs (including a critical division-by-zero in the federation coordinator), and 35 compliance gaps (missing "RESEARCH USE ONLY" disclaimers). All fixes are non-breaking, maintain existing APIs, and pass ruff lint + format checks.

## Features

- **Security Hardening:** Replaced all `torch.load(weights_only=False)` with `weights_only=True` to prevent pickle-based arbitrary code execution. Added `allow_pickle=False` to `np.load()` calls. Migrated 3 plain SHA-256 hash operations to HMAC-SHA256. Fixed mutable audit log references in 3 modules. Replaced weak hardcoded pseudonymization salt with `os.urandom(32)`.

- **Logic Bug Fixes:** Fixed critical division-by-zero in federation coordinator weight computation and SCAFFOLD update. Fixed truthiness bugs where zero was incorrectly treated as invalid (dose calculator, patient model). Fixed dead data in multi-sensor fusion anomaly records. Added null-safety checks for NumPy function availability.

- **Compliance Enforcement:** Added "RESEARCH USE ONLY" disclaimer to all 82 Python modules (35 were missing). Every module docstring now contains the regulatory disclaimer required for research-grade clinical software.

- **SECURITY_AUDIT.md:** Created comprehensive audit report documenting methodology, all 61 findings with file/line references, severity ratings, fix descriptions, statistics by category/severity, and ongoing security recommendations.
