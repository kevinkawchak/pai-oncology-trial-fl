# v0.9.9 - Peer Review Hardening: All v0.9.8 Recommendations Implemented

## Summary

v0.9.9 implements all 10 recommended code fixes from the v0.9.8 senior peer review (3x P1 critical, 4x P2 important, 3x P3 enhancement). Changes focus on secrets management, audit integrity, PHI-safe logging, typed exception handling, timestamp normalization, deprecation lifecycle, property-based testing, centralized configuration, structured error codes, and CI security gates. A new `utils/` package provides shared infrastructure for cryptographic key loading, timestamp formatting, log sanitization, configuration management, and error code definitions.

## Features

- **Centralized HMAC key management** (`utils/crypto.py`) — replaced 8 hardcoded cryptographic key literals with environment-driven loading via `PAI_HMAC_KEY`.
- **Audit hash chaining** (`privacy/audit_logger.py`) — added `previous_hash` field and chain verification to `AuditEvent` / `AuditLogger` for tamper-evident event ordering.
- **PHI-safe log sanitizer** (`utils/log_sanitizer.py`) — strips SSN, phone, email, MRN, IP, and date patterns from error logs before emission.
- **Typed exception handlers** — replaced 11 remaining `except Exception` blocks with specific exception tuples across 7 files.
- **UTC timestamp utility** (`utils/time.py`) — `utc_now_iso()` and `utc_now_date()` for consistent ISO 8601 UTC formatting.
- **Legacy deprecation wrappers** — added functional `warnings.warn()` calls to `privacy/phi_detector.py` and `privacy/deidentification.py`.
- **Property-based/fuzz tests** (`tests/test_privacy/test_phi_fuzz.py`) — 6 Hypothesis-driven tests for PHI detection edge cases.
- **Centralized configuration** (`utils/config.py`) — `SecurityConfig` and `OperationalConfig` frozen dataclasses with environment loading.
- **Structured error codes** (`utils/error_codes.py`) — 18 machine-oriented error codes across 6 categories.
- **SEC-006 CI gate** (`scripts/security_scan.py`) — detects hardcoded secret-like literals with inline suppression support.
