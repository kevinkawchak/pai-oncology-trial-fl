Release title
v0.9.7 - Peer Review Fixes Implementation (v0.9.6 Recommendations)

## Summary

v0.9.7 addresses all 9 recommendations from the v0.9.6 senior peer review. Fixes span 23 modified files and 3 new utility scripts across 4 categories: timestamp consistency (P1), typed exception handling (P1/P2), hash algorithm compliance (P2), and developer tooling (P2/P3). All changes pass `ruff check`, `ruff format --check`, version-alignment verification, and static security pattern scanning with zero findings.

## Features

- **UTC Timestamp Hardening (R1)** — Replaced 24 `datetime.now()` calls with `datetime.now(timezone.utc)` across all 6 regulatory-submissions modules, ensuring consistent cross-site timestamps for federated learning and regulatory audit trails.

- **Typed Exception Handlers (R2, R6, R7)** — Narrowed 17 broad `except Exception` handlers to specific exception tuples (`OSError`, `ValueError`, `RuntimeError`, `TypeError`, etc.) across 12 files spanning conversion pipeline, benchmark runner, framework detector, verify_installation, and all 5 CLI tools.

- **MD5 to SHA-256 Migration (R3)** — Replaced `hashlib.md5()` with `hashlib.sha256()` in the RAG compliance agent's embedding generator, aligning with the project-wide SHA-256 policy.

- **Version Alignment CI Check (R4)** — New `scripts/check_version_alignment.py` (71 LOC) validates that `pyproject.toml`, `CITATION.cff`, and `README.md` badge versions are consistent. Integrated into CI and local lint workflow.

- **Privacy Module Deprecation Notices (R5)** — Added `.. deprecated::` docstrings to legacy `privacy/phi_detector.py` and `privacy/deidentification.py` directing new code to canonical sub-package implementations.

- **Static Security Pattern Scanner (R8)** — New `scripts/security_scan.py` (96 LOC) checks for 5 vulnerability patterns (torch.load, np.load, MD5, UTC-naive datetime, bare except) across all Python files. Integrated into local lint workflow.

- **Release Metrics Generator (R9)** — New `scripts/release_metrics.py` (89 LOC) computes Python file counts, LOC, test files, Markdown, YAML, and directory metrics. Supports `--json` for machine-readable output.
