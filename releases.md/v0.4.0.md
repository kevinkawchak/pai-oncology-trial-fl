v0.4.0 - Core Domain Modules and CLI Tools

## Summary

Builds the core domain modules and command-line tools for the PAI Oncology Trial FL platform. Introduces the digital-twins primary domain with patient modeling, treatment simulation, and clinical integration modules, alongside five production-oriented CLI tools for DICOM inspection, dose calculation, trial site monitoring, simulation job management, and deployment readiness validation.

## Features

- **Patient Modeling** (`digital-twins/patient-modeling/patient_model.py`) — Patient digital twin modeling engine with 6+ tumor type enums (NSCLC, SCLC, BREAST_DUCTAL, BREAST_LOBULAR, COLORECTAL, PANCREATIC), 4 growth model strategies (exponential, logistic, Gompertz, von Bertalanffy), patient status tracking, tumor profile / biomarker / organ function dataclasses, physiology engine with growth kinetics and biomarker evolution, and a configurable twin factory
- **Treatment Simulation** (`digital-twins/treatment-simulation/treatment_simulator.py`) — Multi-modality treatment simulation with pharmacokinetic/pharmacodynamic modeling for chemotherapy, radiation therapy, immunotherapy, targeted therapy, and combination protocols. RECIST response categorization, CTCAE toxicity grading (grades 0–5), and all clinical parameters bounded
- **Clinical Integration** (`digital-twins/clinical-integration/clinical_integrator.py`) — Clinical system integration layer supporting EHR, PACS, LIS, RIS, and TPS connections. HL7 FHIR / DICOM / CSV / JSON data format handling, sync status tracking, clinical report generation, and a federated clinical bridge connecting digital twins to the federated learning pipeline
- **DICOM Inspector** (`tools/dicom-inspector/dicom_inspector.py`) — CLI tool for DICOM medical image inspection with argparse subcommands (inspect, validate, summarize), modality-aware analysis, conditional pydicom import, and JSON output
- **Dose Calculator** (`tools/dose-calculator/dose_calculator.py`) — Radiation and chemotherapy dose calculator with reference constants from NCCN guidelines and RTOG protocols, BSA-based dosing, BED/EQD2 calculations, and bounded clinical parameters
- **Trial Site Monitor** (`tools/trial-site-monitor/trial_site_monitor.py`) — Multi-site monitoring dashboard with GREEN/YELLOW/RED status indicators for enrollment, data quality, and compliance across federated trial sites
- **Simulation Job Runner** (`tools/sim-job-runner/sim_job_runner.py`) — Cross-framework simulation job launcher supporting Isaac Sim, MuJoCo, PyBullet, and Gazebo with automatic framework detection
- **Deployment Readiness Checker** (`tools/deployment-readiness/deployment_readiness.py`) — Checklist-based deployment validation with PASS/FAIL/WARNING/REQUIRES_VERIFICATION states across model validation, safety compliance, regulatory, infrastructure, and documentation categories
