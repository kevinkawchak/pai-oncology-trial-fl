v0.3.0 - Repository Foundation, Community Health, and CI/CD Infrastructure

## Summary

Establishes the complete repository foundation for the PAI Oncology Trial FL platform, consolidating initial structure, cooperation model, community health standards, and CI/CD infrastructure into a unified release. Adds the cross-framework unification layer, Q1 2026 standards objectives, framework integration guides, learning domain directories, comprehensive training configuration, and full community health documentation.

## Features

- **Unification Framework** (`unification/`) — Cross-framework interoperability layer with simulation physics bridge (Isaac Sim ↔ MuJoCo bidirectional state conversion), agentic generative AI unified interface (CrewAI/LangGraph/AutoGen/Custom with MCP/OpenAI/Anthropic tool format conversion), surgical robotics integration, cross-platform tools (framework detector, model converter, policy exporter, validation suite), standards protocols, and integration workflow templates
- **Q1 2026 Standards** (`q1-2026-standards/`) — Three standards objectives: model format conversion pipeline, federated model registry with validation, and cross-platform benchmark runner with implementation guide and compliance checklist
- **Framework Integration Guides** (`frameworks/`) — Detailed integration documentation for NVIDIA Isaac Sim/Lab, MuJoCo, Gazebo (ROS 2), and PyBullet with system requirements, installation, version-specific configuration, and code snippets
- **Learning Domain Directories** — Strengths, limitations, and results documentation for supervised learning, reinforcement learning, self-supervised learning, and generative AI applied to oncology clinical trials
- **Training Configuration** (`configs/training_config.yaml`) — Comprehensive training config with environment settings, domain randomization (tissue/instrument/visual/patient variability), PPO/SAC algorithm configs, federated learning parameters, network architecture, safety constraints (force/velocity/workspace limits), curriculum learning, evaluation metrics, deployment settings, and sim-to-real transfer
- **Installation Verification** (`scripts/verify_installation.py`) — importlib-based framework detection with version validation, colored terminal output, and graceful handling of missing optional dependencies
- **Community Health** — CODE_OF_CONDUCT.md (Contributor Covenant v2.1 with clinical context), CONTRIBUTING.md (contribution types, recency/oncology/reproducibility requirements), SECURITY.md (vulnerability reporting SLA, deployer responsibilities), SUPPORT.md (upstream framework support table)
- **GitHub Templates** — Pull request template (7 change type categories, safety/compliance checklist), bug report template (environment, reproduction steps), feature request template (domain category checkboxes, clinical relevance)
- **CI/CD Enhancements** — Added yamllint for YAML validation, validate-scripts job for Python compilation checks, explicit permissions (contents: read), ruff output-format=github for annotation integration
- **Ruff Configuration** (`ruff.toml`) — Standalone ruff config with line-length 120, per-file-ignores for all directories using conditional imports (F401, F402, F821)
- **Requirements Update** — Expanded requirements.txt with pinned versions, release dates, GitHub URLs, grouped by Core/Deep Learning/Medical Imaging/Agentic AI/Visualization/Deployment/Testing
- **Root Files** — Updated README.md with badges, responsible-use notice, repository structure tree, Core Technologies table, Unification Framework section, Multi-Organization Cooperation table, expanded dependency listing. Updated CITATION.cff with additional keywords. Added posts.md placeholder
