## Key Findings

### A) CI Reproducibility Risk (High)
**Observed:**
- CI installs lint tooling with floating versions (`pip install ruff yamllint`).

**Why this can fail checks:**
- Unpinned linter versions can change behavior between runs; a PR can fail in CI even if local checks passed previously.

**Recommended code fix for Claude (do not apply in this review-only PR):**
1. Pin tooling in `.github/workflows/ci.yml`:
   - `ruff==<known-good>`
   - `yamllint==<known-good>`
2. Add a single source of truth for tool versions in `pyproject.toml` optional `dev` dependencies and reference those versions in CI.
3. Add a lightweight diagnostic step in lint job:
   - `python --version`
   - `ruff --version`
   - `yamllint --version`

**Acceptance criteria:**
- `CI / lint-and-format` green on Python 3.10/3.11/3.12 for two consecutive reruns on same commit.

---

### B) Python Matrix Drift Risk (Medium)
**Observed:**
- CI lint matrix spans 3.10–3.12; release toolchain dependencies are not fully locked and may diverge.

**Recommended code fix for Claude:**
1. Add a constraints file (e.g., `constraints/lint.txt`) with pinned lint stack.
2. Use `pip install -c constraints/lint.txt ruff yamllint` in CI.
3. Add a local helper command in docs for developers to reproduce CI lint exactly.

**Acceptance criteria:**
- Local reproducibility command produces same linter versions as CI and no lint differences.

---

### C) Packaging/Metadata Alignment (Medium)
**Observed:**
- `pyproject.toml` is at `0.9.3`; release docs need synchronized `0.9.4` communication.

**Recommended code fix for Claude:**
1. Bump project version to `0.9.4` when implementing functional fixes.
2. Ensure release badge/version references in README/changelog/release notes are aligned in one PR.

**Acceptance criteria:**
- No version skew among `pyproject.toml`, `CHANGELOG.md`, and `releases.md/v0.9.4.md`.

---

### D) Process Hardening for Future PRs (Medium)
**Observed:**
- CI workflow is functional, but lacks explicit fast-fail visibility when lint environment changes.

**Recommended code fix for Claude:**
1. Add `pip freeze | rg 'ruff|yamllint'` output in lint job logs for auditability.
2. Add pre-commit config pinned to the same lint versions.
3. Add a `make lint` (or equivalent script) that mirrors CI commands exactly.

**Acceptance criteria:**
- Contributors can run one command locally that is bit-for-bit compatible with lint-and-format CI behavior.
