## Recommendation-by-Recommendation Disposition

### P1-1 (Critical): Remove hardcoded cryptographic keys

**Status:** FIXED

**Problem:** Multiple modules embedded static HMAC keys directly in source code (`_HMAC_KEY = b"..."` literals), creating predictable signatures and key leakage risk.

**Fix:**
- Created `utils/crypto.py` with `get_hmac_key()` that loads the HMAC key from the `PAI_HMAC_KEY` environment variable, falling back to a development default only when the variable is unset.
- Replaced all 8 hardcoded key literals across the codebase with `from utils.crypto import get_hmac_key` and `_HMAC_KEY = get_hmac_key()`.

**Files changed (8):**

| File | Old key literal |
|------|----------------|
| `privacy/audit_logger.py` | `b"pai-oncology-audit-integrity-key"` |
| `clinical-analytics/survival_analysis.py` | `b"pai-oncology-trial-fl-default-key"` |
| `clinical-analytics/consortium_reporting.py` | `b"pai-consortium-audit-integrity-key"` |
| `clinical-analytics/pkpd_engine.py` | `b"pai-oncology-pkpd-audit-key"` |
| `clinical-analytics/risk_stratification.py` | `b"pai-oncology-risk-stratification-key"` |
| `clinical-analytics/trial_data_manager.py` | `b"pai-oncology-trial-data-integrity-key"` |
| `clinical-analytics/analytics_orchestrator.py` | `b"pai-oncology-trial-fl-analytics-v0.9.0"` |
| `clinical-analytics/clinical_interoperability.py` | `b"pai-oncology-interop-v0.9.0"` |

**Verification:** All 8 hardcoded key assignments replaced. `PAI_HMAC_KEY` env var controls key in production.

### P1-2 (Critical): Strengthen audit integrity chain

**Status:** FIXED

**Problem:** Audit integrity hashes were per-event but not chained to prior event hash, so event deletion/reordering was not detectable.

**Fix:**
- Added `previous_hash: str = ""` field to `AuditEvent` dataclass.
- Updated `_compute_hash()` to include `previous_hash` in the HMAC payload: `HMAC(previous_hash:event_type:actor:resource:action:timestamp)`.
- Updated `AuditLogger.log()` to thread `previous_hash = events[-1].integrity_hash` into each new event.
- Updated `verify_integrity()` to verify both chain-link consistency (previous_hash matches prior event) and hash recomputation.
- Updated `to_dict()` to persist `previous_hash`.

**Files changed (1):**

| File | Changes |
|------|---------|
| `privacy/audit_logger.py` | +16 / -4 lines |

### P1-3 (Critical): Add PHI-safe logging guardrails

**Status:** FIXED

**Problem:** Error paths logged raw exception text that could leak PHI values.

**Fix:**
- Created `utils/log_sanitizer.py` with `sanitize_log_message()` that scrubs 6 PHI pattern types (SSN, phone, email, MRN, IP, dates) from log strings, replacing matches with `[REDACTED-<type>]`.
- Updated `deidentification_pipeline.py` to wrap exception message in `sanitize_log_message(str(exc))`.
- Updated `phi_detector.py` (canonical) to wrap both DICOM and Presidio error handlers.

**Status:** FIXED
