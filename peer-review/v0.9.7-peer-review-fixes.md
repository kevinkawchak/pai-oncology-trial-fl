### R1 (P1): Harden UTC-aware timestamps in regulatory-submissions

**Status:** FIXED

**Problem:** All 6 regulatory-submissions modules used `datetime.now()` (local-time-naive), creating timestamp inconsistency across distributed federated learning sites and regulatory submissions.

**Fix:**
- Added `timezone` to `from datetime import ...` in all 6 files.

### R2 (P1): Narrow broad exception handlers to typed errors

**Status:** FIXED

**Problem:** Multiple files used bare `except Exception as exc:` handlers, which swallow unexpected errors (e.g., `KeyboardInterrupt`, `SystemExit`) and obscure root causes during debugging.

**Fix:** Replaced every broad `except Exception as exc:` with a tuple of specific exception types appropriate to each context:

| File | Location | Old | New |
|------|----------|-----|-----|
| `unification/cross_platform_tools/framework_detector.py` | `_detect_python_package()` | `except Exception as exc` | `except (AttributeError, TypeError, OSError) as exc` |
| `q1-2026-standards/objective-1-model-conversion/conversion_pipeline.py` | `run_conversion()` | `except Exception as exc` | `except (OSError, ValueError, RuntimeError, TypeError) as exc` |
| `q1-2026-standards/objective-1-model-conversion/conversion_pipeline.py` | `verify()` | `except Exception as exc` | `except (OSError, ValueError, RuntimeError, TypeError) as exc` |
| `q1-2026-standards/objective-1-model-conversion/conversion_pipeline.py` | `_verify_roundtrip()` | `except Exception as exc` | `except (ConversionError, VerificationError, OSError, ValueError, RuntimeError) as exc` |
| `q1-2026-standards/objective-3-benchmarking/benchmark_runner.py` | `run()` | `except Exception as exc` | `except (OSError, ValueError, RuntimeError, TypeError) as exc` |
| `q1-2026-standards/objective-3-benchmarking/benchmark_runner.py` | `run_accuracy()` | `except Exception as exc` | `except (OSError, ValueError, RuntimeError, TypeError) as exc` |
| `q1-2026-standards/objective-3-benchmarking/benchmark_runner.py` | `run_memory()` | `except Exception as exc` | `except (OSError, ValueError, RuntimeError, TypeError) as exc` |
| **Total** | | **7 handlers narrowed** | |

### R3 (P2): Replace MD5 hash with SHA-256 in RAG embedding generator

**Status:** FIXED

**Problem:** `06_oncology_rag_compliance_agent.py` used `hashlib.md5()` for synthetic word embedding generation. While not cryptographic, MD5 is flagged by security scanners and contradicts the project-wide SHA-256 policy.

**Fix:** Changed line 661 from `hashlib.md5(word.encode())` to `hashlib.sha256(word.encode())`.

**File:** `agentic-ai/examples-agentic-ai/06_oncology_rag_compliance_agent.py` (1 instance)

### R5 (P3): Consolidate duplicate privacy module implementations

**Status:** ADDRESSED (deprecation notices added)

**Problem:** Two parallel PHI detection implementations exist:
- `privacy/phi_detector.py` — legacy regex-only detector (v0.1.0)
- `privacy/phi-pii-management/phi_detector.py` — canonical full-featured detector (v0.6.0)

Similarly for de-identification:
- `privacy/deidentification.py` — legacy pipeline (v0.1.0)
- `privacy/de-identification/deidentification_pipeline.py` — canonical pipeline (v0.6.0)

**Fix:**
- Added `.. deprecated::` docstring notices to both legacy modules (`privacy/phi_detector.py` and `privacy/deidentification.py`) directing new code to the canonical sub-package implementations.
- The legacy modules are retained because other modules (`privacy/deidentification.py`, test files) import from them. A full removal requires a coordinated migration across 10+ dependent files which is tracked for a future release.

### R6 (P2): Improve verify_installation.py exception reporting

**Status:** FIXED

**Problem:** `scripts/verify_installation.py` used `except Exception as exc` in `_check_package()`, providing no exception type information in error messages and catching overly broad errors.

**Fix:**
- Narrowed `except Exception as exc` to `except (AttributeError, TypeError, OSError, ValueError) as exc` with improved message format including `type(exc).__name__`.
- Narrowed inner metadata fallback from `except Exception` to `except (ImportError, ValueError)`.
