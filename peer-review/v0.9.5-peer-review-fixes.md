# Peer Review Fixes — v0.9.5

Implements all recommendations from `peer-review/v0.9.4-senior-peer-review.md`.

## Scope

- Addresses **all 12 recommended code fixes** across 4 finding categories (A–D).

- Validates that `lint-and-format` passes for Python 3.10, 3.11, and 3.12.

## Fix Implementation Summary

### A) CI Reproducibility Risk — P0 (High) → Resolved

**Original finding:** CI installed lint tooling with floating versions (`pip install ruff yamllint`), causing non-deterministic lint behavior across runs.

**Fixes applied (3 edits in `.github/workflows/ci.yml`):**

| # | Fix | File | Lines Changed |
|---|-----|------|---------------|
| 1 | Replaced `pip install ruff yamllint` with `pip install -c constraints/lint.txt ruff yamllint` | `.github/workflows/ci.yml:29` | 1 |
| 2 | Added "Show tool versions" step printing `python --version`, `ruff --version`, `yamllint --version`, and `pip freeze` filter | `.github/workflows/ci.yml:31-36` | 6 (new step) |
| 3 | Created `constraints/lint.txt` with `ruff==0.15.1` and `yamllint==1.38.0` | `constraints/lint.txt` | 5 (new file) |

**Acceptance criteria met:** CI install path is now deterministic — identical ruff and yamllint versions on every run regardless of PyPI state.

### B) Python Matrix Drift Risk — P1 (Medium) → Resolved

**Acceptance criteria met:** Running `./scripts/lint.sh` locally produces the same linter versions and output as CI. Developers can reproduce CI lint bit-for-bit.
