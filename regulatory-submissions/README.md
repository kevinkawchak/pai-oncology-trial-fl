## Overview

The `regulatory-submissions/` module provides end-to-end regulatory submission management for
AI/ML-enabled medical device and drug submissions across FDA, EMA, PMDA, and other global
regulatory authorities. It automates the compilation, validation, tracking, and analytics of
regulatory submission packages — from initial IND filings through 510(k)/PMA/De Novo device
clearances — while enforcing 21 CFR Part 11 compliance, ICH guidelines, and ISO/IEC standards
throughout the workflow.

All submission artifacts are generated as structured Markdown or in-memory data structures.
No network calls, external APIs, or file-system writes are performed — the module operates
entirely in-process for research and simulation purposes.

## Relationship to clinical-analytics/

The `regulatory-submissions/` and `clinical-analytics/` modules are complementary halves of the
trial lifecycle pipeline:

| Concern | `clinical-analytics/` | `regulatory-submissions/` |
|---|---|---|
| **Primary focus** | Upstream analytics (PK/PD modeling, survival analysis, risk stratification, data harmonization) | Downstream regulatory packaging (eCTD compilation, submission tracking, compliance validation, regulatory intelligence) |
| **Data flow direction** | Raw trial data → analytic results | Analytic results → submission-ready documents |
| **Regulatory standards** | ICH E9(R1), CDISC ADaM/SDTM | 21 CFR Part 11, eCTD v4, ICH CTD, ISO 14971, IEC 62304 |
| **Privacy model** | Federated computation, differential privacy, secure aggregation | Audit trail, access control, document integrity (SHA-256) |
| **Key outputs** | Survival curves, PK parameters, risk scores, harmonized datasets | eCTD module structures, compliance reports, submission manifests, regulatory timelines |

### Integration points

1. **Survival analysis → Clinical Study Report (CSR)**: Kaplan-Meier estimates and Cox PH
   results from `survival_analysis.py` feed into CSR synopsis generation via
   `document_generator.py`.
2. **Risk stratification → Predetermined Change Control Plan (PCCP)**: Risk tier definitions
   and model performance metrics from `risk_stratification.py` inform the AI/ML PCCP
   document generated by `document_generator.py`.
3. **PK/PD modeling → Nonclinical/Clinical summaries**: Population PK parameters from
   `pkpd_engine.py` are referenced in eCTD Module 2 and Module 5 sections compiled by
   `ectd_compiler.py`.
4. **Data harmonization → Compliance validation**: CDISC conformance results from
   `clinical_interoperability.py` are validated against regulatory requirements by
   `compliance_validator.py`.
5. **Consortium reporting → Submission analytics**: Multi-site reporting metrics from
   `consortium_reporting.py` contribute to submission KPI tracking in
   `submission_analytics.py`.

Both modules share the same architectural principles: stdlib + numpy + scipy only, structured
logging, dataclass-driven APIs, privacy-by-design, and comprehensive test coverage.

## Quick Start

```python
from regulatory_submissions.submission_orchestrator import (
    RegulatorySubmissionOrchestrator,
    SubmissionConfig,
    SubmissionType,
)

config = SubmissionConfig(
    submission_id="SUB-2026-001",
    submission_type=SubmissionType.ECTD_510K,
    sponsor_name="PAI Oncology Research",
    device_name="AI-Assisted Tumor Detection v2.0",
)
orchestrator = RegulatorySubmissionOrchestrator(config)
manifest = orchestrator.compile_manifest()
```
